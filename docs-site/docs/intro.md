# AAG 产品介绍

## 一、产品概述

**AAG（Analytics Augmented Generation Engine）**
是一款面向复杂关系与网络数据分析场景的 **分析增强生成（Analytics Augmented Generation）产品**。
它通过将
**图分析能力、知识增强检索（RAG）与大语言模型（LLM）深度融合**，帮助用户从海量复杂数据中获得
**可解释、可追溯、可落地的分析结论**。

AAG 不只是一个"会回答问题的模型"，而是一个
**能理解业务关系、执行专业分析、并生成决策级结论的智能分析引擎**。

------------------------------------------------------------------------

## 二、产品定位

-   🎯 **定位**：面向图结构与关系型数据的智能分析与生成引擎
-   👥 **目标用户**：风控分析师、数据科学家、研究人员、企业决策支持团队
-   🧠 **核心价值**：让复杂关系分析像"对话"一样简单

------------------------------------------------------------------------

## 三、核心能力

### 3.1 分析增强生成（AAG）能力

AAG 在传统 RAG 基础上引入 **图分析与算法推理能力**，实现：

> **"检索 + 分析 + 推理 + 生成" 的闭环增强**

支持从自然语言问题出发，自动完成：

1.  业务问题理解
2.  图结构建模
3.  图算法分析
4.  结果总结与报告生成

------------------------------------------------------------------------

### 3.2 图计算与关系分析引擎

AAG 内置高性能图计算与关系分析引擎，面向复杂关系网络提供系统化、可扩展的分析能力：

- **丰富的图分析算法体系**  
  支持 PageRank、连通分量、各类中心性分析，以及社区发现、路径分析与关系溯源等核心能力

- **灵活的图查询能力**  
  支持节点查询、路径查询与关系筛选，便于快速定位关键实体与关系结构

- **成熟的图计算基础**  
  基于 NetworkX 与 Neo4j 构建，兼顾算法灵活性与工程稳定性

- **面向规模化场景设计**  
  支持大规模关系网络的高效分析与扩展，满足生产级应用需求

------------------------------------------------------------------------

### 3.3 混合检索（Hybrid RAG）

AAG 采用 **图检索 + 向量检索** 的混合机制：

-   **图检索**：在关系网络中定位关键子图（NebulaGraph）
-   **向量检索**：基于语义相似度召回背景知识（Milvus）
-   **知识融合**：多源信息统一注入 LLM 推理过程

👉 相比单一 RAG，分析结果 **更精准、更具业务语义**

------------------------------------------------------------------------

### 3.4 智能问题建模

-   自动将自然语言业务问题转化为图分析任务
-   支持多步骤、多依赖的复杂问题建模
-   内置任务分解与策略规划能力

------------------------------------------------------------------------

### 3.5 多任务流水线与自动编排

-   自动识别分析任务间的依赖关系
-   构建任务执行 DAG（有向无环图）
-   支持并行执行，提高整体分析效率

------------------------------------------------------------------------

## 四、数据与系统灵活性

### 4.1 多类型数据支持

-   **图数据**：已有关系网络、知识图谱
-   **表格数据**：CSV / Excel 业务数据
-   **文本数据**：报告、日志、非结构化文本

支持从"零数据"到"图建模"的自动化构建流程。

------------------------------------------------------------------------

### 4.2 多种运行模式

-   **交互式模式**：对话式分析，适合探索与研判
-   **批处理模式**：自动化分析任务，适合生产环境

------------------------------------------------------------------------

### 4.3 配置驱动架构

-   通过 YAML 文件配置模型、算法、数据源
-   快速适配不同业务场景与部署环境
-   无需频繁修改代码

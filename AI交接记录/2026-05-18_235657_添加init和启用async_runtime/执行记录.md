## 执行过程

### 任务1：添加 __init__.py
1. 创建 `web/__init__.py`（空文件）→ 使 `web/` 成为 Python 包
2. 创建 `web/frontend/__init__.py`（空文件）→ 使 `web/frontend/` 成为 Python 包
3. 评估相对导入可行性：
   - `routes_manage_dataset.py` 位于 `web.frontend.route`，需导入顶级包 `aag`
   - Python 相对导入 `from ....aag.api.DocumentAPI import server_Test` 会越出顶级包边界 → `ValueError`
   - **结论**：sys.path hack 必须保留。`aag` 与 `web` 是同级顶级包，非父子关系
4. 更新两个文件的注释，将过时的 TODO（"添加 __init__.py 后改为相对导入"）替换为准确说明（解释为什么相对导入不可行）

### 任务2：启用 async_runtime
1. 取消注释 `from aag.api.async_runtime import start_async_runtime, stop_async_runtime`
2. 在 `create_app()` 中：
   - 取消注释 `start_async_runtime()` 调用
   - 包装在 try/except 中，防止启动失败阻塞 Flask 应用启动
   - 成功时记录 `logger.info("异步运行时已启动...")`
   - 失败时记录 `logger.error("异步运行时启动失败: %s...")`，路由层已有 fallback 到 `asyncio.run()`
   - 取消注释 `atexit.register(stop_async_runtime)`（stop 函数内部有 `if not background_loop: return` 守卫，即使 start 失败也安全）

### 分析：`start_async_runtime()` 的异常行为
源自 `aag/api/async_runtime.py:76-77`：
```python
fut = asyncio.run_coroutine_threadsafe(_initialize_components(), background_loop)
fut.result()  # 若 _initialize_components 抛出异常，此处向上传播
```
因此 `create_app()` 中的 try/except 可以正确捕获初始化失败，记录日志但不阻止应用启动。
